\chapter{Lean コードの読み方}
\label{chap:ch01_reading_code}

\begin{chapterabstract}
本章では、Lean 4 のコードを「書く」前に、まず「読む」ことに集中します。
Lean のコードには、\texttt{:}、\texttt{:=}、引数、関数適用といった、最初に意味を取り違えやすい記号が現れます。
本章では、それらを一つずつ取り上げます。
\texttt{\#check} で型を確かめ、\texttt{\#eval} で結果を確かめながら、「この式は何を表し、何に評価されるのか」を上から下に読めるようにします。
命題や証明は、まだ扱いません。
\end{chapterabstract}

\begin{learninggoals}
\item \texttt{:} を「左の式を右の型として読む」注釈として読めます。
\item \texttt{:=} を「右の内容で名前を定義する」記号として読めます。
\item 関数適用が空白で書かれることと、\texttt{f x} が「\texttt{f} を \texttt{x} に適用する」ことを読めます。
\item \texttt{\#check} で式の型を、\texttt{\#eval} で式の評価結果を確認できます。
\end{learninggoals}

\section{この章を読む理由}

プログラミング言語を新しく学ぶとき、最初の障害は構文そのものです。
とくに Lean では、見慣れた記号が、ふだんと少し違う意味を持つことがあります。
たとえば、\texttt{:} は型注釈、\texttt{:=} は定義を表し、関数呼び出しには括弧を使いません。
これらを最初に正しく読めるようにしておくと、以降の章のコードを読みやすくなります。

\section{\texttt{:} は型注釈である}

Lean では、すべての式に型があります。
式がどの型を持つかを明示したいときは、\texttt{(式 : 型)} と書きます。
この \texttt{:} は「左の式を、右の型として読む」という注釈であり、代入でも変換でもありません。

式の型を確認するには、\texttt{\#check} を使います。
\texttt{\#check} は、「この式の型は何か」を Lean に尋ねるコマンドです。

\begin{listing}[htbp]
\begin{minted}{lean4}
#check (3 : Nat)
#check (true : Bool)
#check ("lean" : String)
\end{minted}
\caption{\texttt{:} と \texttt{\#check}}
\label{lst:ch01_check}
\end{listing}

コード\ref{lst:ch01_check}では、\texttt{3} を \texttt{Nat}、\texttt{true} を \texttt{Bool}、\texttt{"lean"} を \texttt{String} として読むよう注釈しています。
\texttt{\#check (3 : Nat)} を実行すると、Lean は \texttt{3 : Nat} と返します。
これは「\texttt{3} は \texttt{Nat} 型です」という意味です。

\section{\texttt{\#eval} は式を評価する}

\texttt{\#check} が型を尋ねるコマンドなのに対し、\texttt{\#eval} は式を実際に計算して結果を表示するコマンドです。
型を確認するときは \texttt{\#check}、結果を確認するときは \texttt{\#eval} と使い分けます。

\begin{listing}[htbp]
\begin{minted}{lean4}
#eval (3 : Nat)
#eval 2 + 3
#eval "lean" ++ "4"
\end{minted}
\caption{\texttt{\#eval}}
\label{lst:ch01_eval}
\end{listing}

コード\ref{lst:ch01_eval}では、\texttt{2 + 3} は \texttt{5} に、\texttt{"lean" ++ "4"} は \texttt{"lean4"} に評価されます。
\texttt{++} は文字列の連結です。
\texttt{\#eval} は計算結果を表示するだけであり、何かを証明するものではありません。

\begin{leanbox}
\texttt{\#check} と \texttt{\#eval} は、どちらも \texttt{\#} で始まる確認用コマンドです。
\texttt{\#check} は型を、\texttt{\#eval} は値を表示します。
本書ではコードの意味を確かめるために、この二つを繰り返し使います。
\end{leanbox}

\section{\texttt{:=} と関数適用}

名前を定義するときは \texttt{def} を使い、\texttt{:=} の右側に中身を書きます。
この \texttt{:=} は「右の内容でこの名前を定義する」という記号であり、後から書き換える代入とは異なります。
引数を取る関数は、名前の後ろに \texttt{(引数 : 型)} を並べて書きます。

定義した関数を呼び出すときは、括弧ではなく空白を使います。
\texttt{addOne 5} は、「\texttt{addOne} という関数を \texttt{5} に適用する」と読みます。

\begin{listing}[htbp]
\begin{minted}{lean4}
def greeting : String :=
  "hello"

def addOne (n : Nat) : Nat :=
  n + 1

#check addOne
#eval greeting
#eval addOne 5
\end{minted}
\caption{\texttt{greeting} と \texttt{addOne}}
\label{lst:ch01_def}
\end{listing}

コード\ref{lst:ch01_def}では、\texttt{greeting} は引数を取らない \texttt{String} 型の定義です。
\texttt{addOne} は \texttt{Nat} を一つ受け取り、\texttt{Nat} を返す関数です。
\texttt{(n : Nat)} は、「引数 \texttt{n} は \texttt{Nat} 型です」と読みます。
\texttt{\#eval addOne 5} は、\texttt{addOne} を \texttt{5} に適用した結果 \texttt{6} を表示します。

\section{本章のまとめ}

\begin{summarybox}
要点と記法
\begin{itemize}
\item \texttt{(式 : 型)}：左の式を右の型として読む型注釈
\item \texttt{def 名前 := 本体}：名前の定義
\item \texttt{f x}：空白で書く関数適用
\item \texttt{\#check}：式の型を表示するコマンド
\item \texttt{\#eval}：式の評価結果を表示するコマンド
\end{itemize}
\end{summarybox}
