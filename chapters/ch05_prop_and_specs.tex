\chapter{\texttt{Prop} と仕様名}
\label{chap:ch05_prop_and_specs}
\BookIndex{Bool}{\texttt{Bool}}
\BookIndex{Prop}{\texttt{Prop}}
\BookIndex{しよう}{仕様}

\begin{chapterabstract}
本章では、これまで具体例で読んできた \texttt{hasDiscount member = true} という形の主張に名前を付けます。
そのために、命題の型 \texttt{Prop} を導入し、\texttt{member} の値ごとに命題を作る \texttt{DiscountSpec} を定義します。
\end{chapterabstract}

\begin{learninggoals}
\item \texttt{Bool} と \texttt{Prop} の違いを、計算される値と証明したい主張の違いとして説明できます。
\item \texttt{Prop} を返す定義を書き、仕様に名前を付けられます。
\item \texttt{DiscountSpec true} と \texttt{DiscountSpec false} が、それぞれどのような命題になるかを展開できます。
\end{learninggoals}

\section{命題に仕様名を付ける}

ここまで、\texttt{hasDiscount true = true} のような主張を何度も読んできました。
同じ形の主張を毎回そのまま書くと冗長になり、設計文書とも対応させにくくなります。
そこで、主張に名前を付けます。
名前を付けると、その主張を「仕様」として扱えるようになり、後から名前で参照できます。

\section{\texttt{Bool} と \texttt{Prop} の違い}

\texttt{hasDiscount member} は \texttt{Bool} を返します。
この返り値は、実行時に \texttt{true} または \texttt{false} に\emph{計算される値}です。
一方、\texttt{hasDiscount member = true} は計算される値ではなく、成り立つかどうかを問う\emph{主張}です。
Lean では、このような主張の型を \texttt{Prop} と書きます。

図\ref{fig:ch05-bool-vs-prop}は、実行時に計算される判定と、その判定について証明する仕様を、別の層として示します。

\FigurePlaceholder{fig-ch05-bool-vs-prop.png}{0.88}{\texttt{Bool} と \texttt{Prop}}{fig:ch05-bool-vs-prop}

\begin{table}[htbp]
\centering
\caption{\texttt{Bool} と \texttt{Prop} の比較}
\label{tab:ch05-bool-prop}
\begin{tabular}{p{0.18\linewidth}p{0.34\linewidth}p{0.34\linewidth}}
\toprule
観点 & \texttt{Bool} & \texttt{Prop} \\
\midrule
役割 & 実行時に計算される真偽値 & 証明したい主張 \\
確認方法 & \texttt{\#eval} で値を見る & \texttt{rfl} などで証明する \\

\end{tabular}

表\ref{tab:ch05-bool-prop}のように、\texttt{Bool} は判定結果として、\texttt{Prop} は保証したい主張として読み分けます。
両者は競合するものではありません。
実務では、実装の判定を \texttt{Bool} で書き、その判定が満たすべき性質を \texttt{Prop} で書きます。

\section{仕様に名前を付ける}

\texttt{member} の値ごとに「割引がある」という命題を作る定義を書きます。

\begin{listing}[htbp]
\begin{minted}{lean4}
def hasDiscount (member : Bool) : Bool :=
  member

def DiscountSpec (member : Bool) : Prop :=
  hasDiscount member = true
\end{minted}
\caption{\texttt{DiscountSpec}}
\label{lst:ch05_discountspec}
\end{listing}

コード\ref{lst:ch05_discountspec}の \texttt{DiscountSpec} は、\texttt{Bool} を受け取って \texttt{Prop} を返す定義です。
つまり、\texttt{member} の値ごとに「\texttt{hasDiscount member = true} が成り立つ」という命題を作ります。
返り値の型が \texttt{Prop} であることは、\texttt{DiscountSpec} が値ではなく主張の定義であることを示します。

\section{名前に値を入れて展開する}

仕様名に具体的な値を入れると、どのような命題になるかが決まります。

\begin{listing}[htbp]
\begin{minted}{lean4}
def hasDiscount (member : Bool) : Bool :=
  member

def DiscountSpec (member : Bool) : Prop :=
  hasDiscount member = true

#check DiscountSpec true
#check DiscountSpec false
\end{minted}
\caption{\texttt{DiscountSpec true}・\texttt{DiscountSpec false}}
\label{lst:ch05_apply}
\end{listing}

コード\ref{lst:ch05_apply}では、\texttt{DiscountSpec true} と \texttt{DiscountSpec false} がそれぞれ命題になっています。
命題の内容は、定義に値を入れて展開すると分かります。

\begin{table}[htbp]
\centering
\caption{\texttt{DiscountSpec true}・\texttt{DiscountSpec false}}
\label{tab:ch05-expand}
\begin{tabular}{lll}
\toprule
書いた式 & 展開後の命題 & 証明できるか \\
\midrule
\texttt{DiscountSpec true} & \texttt{hasDiscount true = true} → \texttt{true = true} & できる \\
\texttt{DiscountSpec false} & \texttt{hasDiscount false = true} → \texttt{false = true} & できない \\
\bottomrule
\end{tabular}
\end{table}

表\ref{tab:ch05-expand}のように、\texttt{DiscountSpec true} は最終的に \texttt{true = true} になり、証明できます。
\texttt{DiscountSpec false} は \texttt{false = true} になり、証明できません。
仕様名を見たときに、この展開を頭の中でたどれることが大切です。

\section{本章のまとめ}

要点と記法
\begin{itemize}
\item \texttt{Bool}：計算される真偽値
\item \texttt{Prop}：証明の対象となる命題の型
\item \texttt{def 名前 (引数) : Prop := 主張}：仕様への命名
\end{itemize}
