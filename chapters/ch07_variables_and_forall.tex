\begin{chapterabstract}
本章では、値を一つに固定しない主張を扱います。
「任意の価格について」「すべてのユーザーについて」という形の仕様を書くために、引数を取る定理、\texttt{forall}、および証明を始める \texttt{intro} を導入します。
さらに、無名関数、型パラメータ、関数合成と、\texttt{Option} と \texttt{match} を使った小さなパイプラインも扱います。
\end{chapterabstract}

\begin{learninggoals}
\item 引数を取る定理で「任意の入力について成り立つ」主張を書けます。
\item \texttt{forall} と \texttt{intro} を使い、すべての値についての仕様を証明できます。
\item 無名関数 \texttt{fun}、型パラメータ、関数合成を読めます。
\item \texttt{Option} と \texttt{match} を使った小さなパイプラインを読めます。
\end{learninggoals}

\section{具体例から任意の入力へ}

テストは、選んだ具体例についての確認です。
一方、仕様で述べたい内容は、しばしば「すべての入力でこの性質が成り立つ」という主張です。
本章では、値を固定しない主張の書き方と、証明の始め方を学びます。
これは、後のマイグレーション証明や圏論の法則確認で何度も使う基本形になります。

\section{任意の値についての定理}

定理が引数を取ると、その引数は「任意の値」を表します。
次の定理は、税額が \texttt{0} のとき、税込金額が元の価格と一致することを任意の \texttt{price} について示します。

\begin{listing}[htbp]
\begin{minted}{lean4}
def addTax (price tax : Nat) : Nat :=
  price + tax

theorem addTax_zero (price : Nat) :
    addTax price 0 = price := by
  rfl
\end{minted}
\caption{\texttt{addTax\_zero}}
\label{lst:ch07_forall_price}
\end{listing}

コード\ref{lst:ch07_forall_price}の \texttt{addTax\_zero} は、特定の価格ではなく、任意の \texttt{price} についての主張です。
\texttt{addTax price 0} は定義により \texttt{price + 0} になり、\texttt{price} に展開されるため、\texttt{rfl} で閉じられます。

\section{\texttt{forall} と \texttt{intro}}

「すべての \texttt{x} について \texttt{P x}」という主張は、\texttt{forall x, P x} と書けます。
Lean では ASCII の \texttt{forall} と Unicode の \texttt{∀} は同じ全称量化を表します。
同様に、\texttt{->} と \texttt{→} は同じ関数型または含意の矢印です。
これを証明するには、任意の値を一つ \texttt{intro} で受け取り、その値について示します。
次の例では、ID を取り出す処理が値を変えないという仕様を扱います。

\begin{listing}[htbp]
\begin{minted}{lean4}
def toDtoId (id : Nat) : Nat :=
  id

def PreservesId (f : Nat -> Nat) : Prop :=
  forall id, f id = id

theorem toDtoId_preservesId : PreservesId toDtoId := by
  intro id
  rfl
\end{minted}
\caption{\texttt{PreservesId}}
\label{lst:ch07_forall_intro}
\end{listing}

コード\ref{lst:ch07_forall_intro}の \texttt{PreservesId} は、「すべての \texttt{id} について \texttt{f id = id}」という仕様です。
\texttt{intro id} で任意の ID を一つ受け取ります。
\texttt{toDtoId} の定義から値がそのまま返るため、\texttt{rfl} で閉じます。

\section{高階関数と関数合成を段階的に読む}

\subsection{無名関数と、関数を受け取る関数}

関数は、その場で名前を付けずに書けます。
\texttt{fun n => n + 1} は、「\texttt{n} を受け取り \texttt{n + 1} を返す関数」です。
まずは \texttt{inc} と \texttt{applyTwice} だけを読み、値の引数と関数の引数を区別します。

\subsection{型パラメータと関数合成}

型を引数に取る関数合成は、出力の型と入力の型が合う関数をつなぎます。

\begin{listing}[htbp]
\begin{minted}{lean4}
def inc : Nat -> Nat :=
  fun n => n + 1

def applyTwice (f : Nat -> Nat) (x : Nat) : Nat :=
  f (f x)

def compose {A B C : Type} (g : B -> C) (f : A -> B) : A -> C :=
  fun x => g (f x)

#eval applyTwice inc 10
#eval compose inc inc 10
\end{minted}
\caption{\texttt{inc}・\texttt{applyTwice}・\texttt{compose}}
\label{lst:ch07_compose}
\end{listing}

コード\ref{lst:ch07_compose}では、\texttt{inc} は無名関数に名前を付けた定義であり、\texttt{applyTwice} は関数を引数に取る関数です。
\texttt{compose} は \texttt{\{A B C : Type\}} という型パラメータを取り、\texttt{compose g f} を「先に \texttt{f}、後で \texttt{g}」と読みます。
波括弧で囲まれた型パラメータは、呼び出し側の型から Lean が推論します。

関数を合成するには、型がつながる必要があります。
\texttt{A -> B} の出力 \texttt{B} が \texttt{B -> C} の入力と一致するため、この二つの関数をつなげられます。
この見方は、関数を「合成できる変換」として捉える基礎になります。

\texttt{compose g f} の途中の値は \texttt{f} の出力なので、型は \texttt{B} です。
\texttt{g} は \texttt{B} を入力に取るため、その値を受け取れます。
一方、最初の入力は \texttt{A} 型なので、一般には \texttt{g} を先に適用できません。

図\ref{fig:ch07-pipeline}は、\texttt{f : A -> B} の出力型 \texttt{B} を \texttt{g : B -> C} の入力型へつなぎ、二つの関数を一つの処理として読む流れを示します。

\section{\texttt{Option} と \texttt{match} の小さなパイプライン}

値が「あるかもしれない／ないかもしれない」ことは、\texttt{Option} で表します。
値がある場合は \texttt{some x}、ない場合は \texttt{none} と書き、\texttt{match} で場合分けします。

\begin{listing}[htbp]
\begin{minted}{lean4}
def surround (s : String) : String :=
  "[" ++ s ++ "]"

def nonempty (s : String) : Option String :=
  if s == "" then none else some s

def decorateIfNonempty (s : String) : Option String :=
  match nonempty s with
  | none => none
  | some value => some (surround value)

def decorateAll (xs : List String) : List (Option String) :=
  xs.map decorateIfNonempty

#eval decorateIfNonempty ""
#eval decorateIfNonempty "lean"
#eval decorateAll ["a", "", "b"]
\end{minted}
\caption{\texttt{decorateIfNonempty} と \texttt{decorateAll}}
\label{lst:ch07_pipeline}
\end{listing}

コード\ref{lst:ch07_pipeline}では、\texttt{nonempty} が空文字列を \texttt{none} に、それ以外を \texttt{some s} に変換します。
\texttt{decorateIfNonempty} は \texttt{match} で \texttt{Option} を場合分けし、値があるときだけ装飾します。
\texttt{decorateAll} は、\texttt{xs.map} で各要素に同じ処理を適用します。

\texttt{decorateIfNonempty "lean"} では、\texttt{nonempty} の結果が \texttt{some "lean"} になり、\texttt{match} は \texttt{some} の枝を選びます。
最終結果は \texttt{some "[lean]"} です。
\texttt{List.map} は、この同じ処理を各要素へ一括適用します。

\section{本章のまとめ}

要点と記法
\begin{itemize}
\item \texttt{forall x, P x} / \texttt{intro x}：全称量化とその導入
\item \texttt{fun x => 本体}：無名関数
\item \texttt{\{A : Type\}}：推論される型パラメータ
\item \texttt{some} / \texttt{none} / \texttt{match} / \texttt{xs.map}：\texttt{Option} と一括適用
\end{itemize}
