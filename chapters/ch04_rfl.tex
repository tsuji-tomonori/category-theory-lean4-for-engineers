\chapter{\texttt{rfl}：定義を展開すれば同じ}
\label{chap:ch04_rfl}
\BookIndex{rfl}{\texttt{rfl}}
\BookIndex{ていぎじょうのひとしさ}{定義上の等しさ}

\begin{chapterabstract}
本章では、最初の証明道具 \texttt{rfl} だけを扱います。
\texttt{rfl} は、左辺と右辺が定義を展開すると同じ形になる等式を閉じる道具です。
前章で読んだ \texttt{hasDiscount true = true} を実際に \texttt{rfl} で証明し、\texttt{hasDiscount false = true} が閉じられない理由も確認します。
\texttt{simp} や \texttt{rw} は、まだ扱いません。
\end{chapterabstract}

\begin{learninggoals}
\item \texttt{example : 命題 := by rfl} の形で、定義どおりの等式を証明できます。
\item \texttt{rfl} を「定義を展開すると左右が同じになる」ときに使えると説明できます。
\item 成り立たない等式では \texttt{rfl} を使えないことを、展開の過程から説明できます。
\end{learninggoals}

\section{主張を証明として確定する}

前章では、\texttt{hasDiscount true = true} を「成り立つ主張」と読みました。
しかし、「成り立ちそう」と判断することと、Lean が成り立つと認めることは異なります。
本章では、その主張を Lean に確かめさせます。
最初に使う証明道具は、\texttt{rfl} 一つだけです。
\texttt{rfl} を使える条件を明確にすることが、後の証明の土台になります。

\section{\texttt{rfl} で成り立つ等式を閉じる}

証明を書くには、命題の後に \texttt{:= by} と書いて証明モードに入り、そこで \texttt{rfl} を使います。
名前を付けない小さな証明は、\texttt{example} で書けます。

\begin{listing}[htbp]
\begin{minted}{lean4}
def hasDiscount (member : Bool) : Bool :=
  member

example : hasDiscount true = true := by
  rfl
\end{minted}
\caption{\texttt{hasDiscount true = true}}
\label{lst:ch04_rfl_success}
\end{listing}

コード\ref{lst:ch04_rfl_success}では、\texttt{hasDiscount true = true} を \texttt{rfl} で証明しています。
この等式を閉じられる理由は、ゴールの展開を順に追うと分かります。

\begin{table}[htbp]
\centering
\caption{\texttt{hasDiscount true = true}}
\label{tab:ch04-success}
\begin{tabular}{ll}
\toprule
段階 & Lean が見ているゴール \\
\midrule
最初のゴール & \texttt{hasDiscount true = true} \\
\texttt{hasDiscount} を展開 & \texttt{true = true} \\
結論 & 左右が同じなので \texttt{rfl} で閉じられる \\
\bottomrule
\end{tabular}
\end{table}

\section{成り立たない等式は \texttt{rfl} で閉じられない}

\texttt{rfl} は、どのような主張でも証明する道具ではありません。
左右が異なる形に展開される等式には使えません。
次のコードは、意図的に成り立たない主張を書いた例です。
証明できないため、該当箇所をコメントにしています。

\begin{listing}[htbp]
\begin{minted}{lean4}
def hasDiscount (member : Bool) : Bool :=
  member

-- これは証明できない（false = true になるため）
-- example : hasDiscount false = true := by
--   rfl
\end{minted}
\caption{\texttt{hasDiscount false = true}}
\label{lst:ch04_rfl_failure}
\end{listing}

コード\ref{lst:ch04_rfl_failure}のコメントを外すと、Lean はエラーを出します。
その理由も、展開を追うと分かります。

\begin{table}[htbp]
\centering
\caption{\texttt{hasDiscount false = true}}
\label{tab:ch04-failure}
\begin{tabular}{ll}
\toprule
段階 & Lean が見ているゴール \\
\midrule
最初のゴール & \texttt{hasDiscount false = true} \\
\texttt{hasDiscount} を展開 & \texttt{false = true} \\
結論 & 左右が違うので \texttt{rfl} では閉じられない \\
\bottomrule
\end{tabular}
\end{table}

\section{数値の計算でも \texttt{rfl} は使える}

\texttt{rfl} は \texttt{Bool} だけでなく、定義どおりに計算が進む等式であれば数値にも使えます。

\begin{listing}[htbp]
\begin{minted}{lean4}
def addTax (price tax : Nat) : Nat :=
  price + tax

example : addTax 100 0 = 100 := by
  rfl
\end{minted}
\caption{\texttt{addTax 100 0 = 100}}
\label{lst:ch04_rfl_addtax}
\end{listing}

コード\ref{lst:ch04_rfl_addtax}では、\texttt{addTax 100 0} は定義により \texttt{100 + 0} になり、\texttt{100} に計算されます。
左右が同じ \texttt{100} になるため、\texttt{rfl} で閉じられます。

\section{本章のまとめ}

要点と記法
\begin{itemize}
\item \texttt{example : P := by ...}：名前のない証明
\item \texttt{by}：証明モードへの入口
\item \texttt{rfl}：定義展開で左右が同じになる等式を閉じるタクティク
\end{itemize}
