この部では、Lean 4 を「数学者のための難しい証明支援系」としてではなく、ソフトウェアエンジニアが仕様を小さく書いて確認するための言語として導入する。Lean はプログラミング言語でもあり、証明を書くための言語でもある。この二面性が、仕様と実装の関係を考えるうえで重要になる。

最初に学ぶのは、型、値、関数である。これは多くのエンジニアにとって馴染みのある概念だが、Lean では型がより中心的な役割を持つ。型は単なるメモリ上の表現ではなく、「この値はどのような約束を満たしているか」を表す枠組みとして使われる。\texttt{Nat}、\texttt{Bool}、\texttt{String}、\texttt{List}、\texttt{Option} のような基本型を使いながら、関数を定義し、評価し、簡単な性質を確認する。

次に、命題と証明を扱う。Lean では、「この関数は常に同じ結果を返す」「この処理の後も不変条件が保たれる」といった主張を \texttt{Prop} として書くことができる。そして、その主張が正しいことを \texttt{theorem} や \texttt{example} として証明する。最初に使う証明道具は、\texttt{rfl}、\texttt{simp}、\texttt{rw}、\texttt{calc} などの小さなものに絞る。ここで目指すのは、証明の専門家になることではなく、「仕様を Lean 上で検査可能な形にする」感覚を身につけることである。

\begin{keywordbox}
\texttt{rfl}、\texttt{simp}、\texttt{rw}、\texttt{calc} は、以後の証明コードで頻繁に出てくる小さな証明用キーワードである。
\begin{itemize}
\item \texttt{rfl} は、定義を展開して両辺が同じ形になる等式を閉じる。
\item \texttt{simp} は、Lean が知っている単純化規則で式を整理する。
\item \texttt{rw} は、証明済みの等式を使って現在のゴールを書き換える。
\item \texttt{calc} は、等式や不等式の変形を段階的に並べる。
\end{itemize}
厳密には \texttt{rfl}、\texttt{simp}、\texttt{rw} はタクティク、\texttt{calc} は構文として説明されることが多い。本書では、コードを読むときの目印としてこれらをキーワード扱いにし、Lean のコード表示でも同じ色で強調する。
\end{keywordbox}

この部では、自然言語の仕様書をいきなり完全に Lean へ移すことはしない。むしろ、仕様の中から小さく独立した性質を切り出す練習をする。税込金額の計算、文字列正規化、残高が負にならない操作など、身近な例を使って、関数・命題・証明の関係を確認する。

この部を読み終えるころには、読者は Lean で小さな関数を書き、その関数について「こういう性質が成り立つ」と主張し、簡単な証明を書けるようになる。これは後続の圏論やマイグレーション証明を学ぶための土台になる。

\section*{この部で扱うこと}

\begin{itemize}
\item Lean 4 の基本的な型と値
\item \texttt{def} による関数定義
\item \texttt{\#eval} による実行確認
\item \texttt{Bool} と \texttt{Prop} の違い
\item \texttt{example}、\texttt{theorem}、\texttt{by} による証明
\item \texttt{rfl}、\texttt{simp}、\texttt{rw}、\texttt{calc} の基本
\item \texttt{structure} によるデータモデル
\item 事前条件、事後条件、不変条件の最小表現
\end{itemize}

\section*{この部で扱わないこと}

\begin{itemize}
\item 高度なタクティク
\item Mathlib の大規模な数学ライブラリ
\item 圏論の抽象的な定義
\item 実装言語から Lean への自動変換
\end{itemize}
