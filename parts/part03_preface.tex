この部では、Leanで仕様を証明するために必要な道具を整理します。
ここでの目的は、定理を多数覚えることではありません。
ソフトウェア開発でよく出会う主張を、どの種類の証明道具で扱えるのか理解することが目的です。

たとえば、リファクタリング前後の処理が同じであることを示す場合、多くは等式推論として表せます。
式を書き換え、不要な処理を消し、定義を展開して、最終的に同じ形へ到達します。
このときに使うのが、\texttt{rw}、\texttt{simp}、\texttt{calc} です。
これらは、数学の証明だけでなく、レビュー可能な変形ログを書く道具としても理解できます。

また、\texttt{Option}、\texttt{Bool}、\texttt{Sum} のような分岐を含む型では、場合分けが必要です。
エラーケースを忘れないことは、実務上も重要です。
Lean の \texttt{cases} は、分岐を網羅的に扱うための道具として機能します。
通常のテストでは代表例だけを見ることが多い一方、証明では型が持つすべての形を分解して確認します。

リストや木構造のように、長さや深さが任意のデータを扱う場合には、帰納法が必要です。
帰納法は難しい数学の技法に見えるかもしれませんが、ソフトウェア上は「空のケースで成り立つ」「一つ追加しても成り立つ」なら「任意の長さで成り立つ」と考える方法です。
バッチ移行で件数が保存されること、全要素が条件を満たすこと、木構造変換でノード数が保存されることなどに使えます。

さらに、関数外延性、Galois接続、Pullback のような、少し抽象的な道具も扱います。
関数外延性は、wrapper や adapter の同値性確認に使えます。
Galois接続は、詳細な情報から公開情報へ移すような抽象化を説明するために使えます。
Pullback は、複数のデータソースが整合する条件、すなわち join や制約付き合成を考えるための言葉になります。

この部を通じて、読者は「どの実務課題にどの証明道具が対応するか」を見分けられるようになります。
これらの道具を具体的な仕様とマイグレーションのサンプルへ適用できる形で整理します。

\section*{この部で扱うこと}

\begin{itemize}
\item 等式推論と書き換え
\item \texttt{simp} による正規化
\item \texttt{calc} による段階的な証明
\item \texttt{cases} による場合分け
\item \texttt{induction} によるリスト・木構造の証明
\item 関数外延性
\item 前順序とGalois接続
\item Pullback と整合制約
\end{itemize}
