# 改稿基準

## 1. 表記

- 字面から意味を推測しにくい漢字は、一般的な表記にする。
- 常用漢字表外の字や読み、本来の意味からずれた当て字を避ける。
- 意味が変わらない範囲で、難しい漢字をひらがなにする。

例: 「漸く」→「ようやく」、「繋ぐ」→「つなぐ」。

## 2. 語彙

- 厳密さが必要な専門語は残し、対象読者が初見で理解できない場合は初出で説明する。
- 格調や新しさのためだけに選ばれた漢語・外来語を、短い一般語へ置き換える。
- 一般語への置換で定義の境界が変わる場合は置換せず、説明を補う。

例: 文脈上同じ意味なら「エスカレーションする」→「上司に報告する」。専門的な「代替キー」は、初出で「主キーとは別に検索へ使うキー」と説明する。

## 3. 文の組み立て

- 修飾語を修飾先の直前に置く。
- 名詞を連ねて係り先を推測させない。
- 「のみ」「だけ」「すべて」など、範囲を変える語の係り先を明示する。
- 複数の動作を「して」「し」でつなぎ過ぎない。順序、条件、対象を補うか、文を分ける。
- 並列する要素では、必要な名詞や動詞を省略しない。
- 一文には中心となる主張を一つ置く。長さだけを理由に分割せず、論理単位で分ける。

例: 「作業マニュアルやチェックリストの更新」→「作業マニュアルの更新とチェックリストの更新」。

## 4. 文脈

- 「これ」「それ」「この」「その」が複数の対象を指し得る場合は、対象の名詞を繰り返す。
- 逆接や話題転換を重ねない。結論までの関係を、必要最小限の接続語で示す。
- 主語や目的語を省略すると別の参加者を指し得る場合は補う。
- 「前述」「上記」「以下」だけに頼らず、参照対象を短く示す。

例: 「それを確認する」→「その変更内容を確認する」。

## 5. 段落

- 一段落では一つの話題を扱う。
- 冒頭で話題または結論を示し、説明、根拠、例、制約を続ける。
- 別の概念、手順、例外へ移る位置で段落を分ける。
- 箇条書きの各項目は、文法上の形と情報の粒度をそろえる。

## 判定質問

各文と各段落について次を確認する。

1. 初見の対象読者が語の意味を推測できるか。
2. 修飾語、限定語、否定、数量の範囲が一つに決まるか。
3. 省略された主語、目的語、動作対象を一つに決められるか。
4. 指示語が指す対象は一つか。
5. 前文との関係は一つか。
6. 段落の話題と役割は一つか。
7. 書き換え後も技術的な真偽条件は同じか。
