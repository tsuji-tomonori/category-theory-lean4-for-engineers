# 明確で一意な日本語を書く

技術的な意味と文書内の表記規約を保ち、読み手が一度で同じ意味に解釈できる日本語へ改稿する。

## 手順

1. 対象範囲を列挙する。依頼の「すべて」が指すファイルを特定し、対象外も記録する。
2. 用語集、品質基準、前後の章を読む。定義語と一般語を区別する。
3. コード、数式、コマンド、識別子、ラベル、参照、引用の意味を変更しない保護対象として扱う。
4. `references/revision-rules.md` の5層を上から順に全文へ適用する。検索置換だけで完了させず、文脈ごとに判断する。
5. 変更後の各文を「誰が・何を・どの条件で・どうするか」という観点で読み直す。複数に解釈できる場合は、名詞を補うか文を分ける。
6. 段落ごとに話題と役割を一つにする。定義、理由、例、注意、結論が混在する場合は分ける。
7. 差分を読み、事実関係、用語、参照関係、文体を確認する。
8. 対象形式の検証を実行する。LaTeX ではビルドし、コード例を別に検証する。

## LaTeX 文書の扱い

- 本文、見出し、キャプション、脚注、表の説明文を改稿対象に含める。
- `lstlisting`、`verbatim`、Lean コード、数式、`\label`、`\ref`、`\cite`、コマンド名、型名、関数名は原則として維持する。
- 定義語を平易語へ置き換えて意味が変わる場合は、初出で短く説明し、その後は定義語を使う。
- 文体と句読点は文書の既存規約に合わせる。

## 全文監査

LaTeX の日本語本文を漏れなく棚卸しするには、次を実行する。

```bash
python3 skills/write-clear-unambiguous-japanese/scripts/audit_tex_prose.py chapters/*.tex
```

出力された全ファイル数、文候補数、要確認箇所を記録する。要確認箇所は機械的な誤りではない。前後を読み、曖昧でなければ変更しない。

## 完了条件

- 列挙した全対象を確認している。
- 難読表記、説明のない難語、複数に係り得る修飾・並列、指示対象不明の表現、接続の蛇行、話題が混在する段落を解消している。
- 保護対象と技術的な主張を維持している。
- 差分レビュー、形式検証、対象範囲に対応したビルドまたはテストが成功している。
