# 第20章 画像生成プロンプト

## fig-ch20-list-induction.png

- 対応箇所: 第20章 20.1「帰納法を『全長さのリストに対する網羅的確認』として読む」
- 目的: 空リストの基底ケースと、先頭に1要素を追加する cons ケースによって、任意長のリストを覆えることを読者に理解させる。
- 画像生成プロンプト:
  日本語技術書向けの16:9横長、白背景のフラットな階段図。左から「[] / nil」「[a1] / cons a1 []」「[a2, a1] / cons a2 (cons a1 [])」「[a3, a2, a1] / cons a3 (cons a2 (cons a1 []))」の4段を並べ、各矢印に「cons で先頭に1要素追加」と置く。最後は「… 任意の長さへ」へ続ける。リスト表記と cons 表記の要素順を必ず一致させる。文字列は指定どおり正確に描き、章番号、図番号、図タイトルは画像内に入れない。余白を広く取り、コード文字が小さくなりすぎないようにする。
- スタイル指定:
  - 日本語技術書向け
  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物
  - 複雑すぎる背景
  - 小さすぎる文字
  - 原典PDFの図に似せすぎること

## fig-ch20-tree-induction.png

- 対応箇所: 第20章 20.5「サンプル：木構造変換でノード数が保存されること」
