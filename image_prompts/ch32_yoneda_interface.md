# 第32章 画像生成プロンプト

## fig-ch32-yoneda-overview.png

- 対応箇所: 第32章「この章を読む理由」
- 目的: 値を直接見るのではなく、複数の観測関数を通じて外から特徴づけるという直感を示す。
- 画像生成プロンプト:
  中央に「値 x : A」と書かれたシンプルな円を置く。円の周囲に複数の小さな窓または関数ボックスを配置し、それぞれ「f : A -> B」「g : A -> C」「h : A -> D」と書く。各関数ボックスから右側に「f x」「g x」「h x」という観測結果の箱を伸ばす。全体として「内部を直接のぞく」のではなく、「外部からの観測結果の集まりで値を見る」構図にする。
- スタイル指定:
  - 日本語技術書向け
  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物
  - 複雑すぎる背景
  - 小さすぎる文字
  - 原典PDFの図に似せすぎること

## fig-ch32-natural-observation.png

- 対応箇所: 第32章「米田の補題を一文で読む」
- 目的: 観測してから後処理する経路と、後処理込みで観測する経路が一致する自然性の雰囲気を示す。
- 画像生成プロンプト:
  左に「A」、中央に「B」、右に「C」の3つの箱を水平に配置する。AからBへ矢印「f」、BからCへ矢印「g」を置く。AからCへ曲線矢印「g ∘ f」を置く。下側に「x」から「f x」へ進み、さらに「g (f x)」へ進む観測結果の流れを描く。別経路として「x」から直接「(g ∘ f) x」へ進む矢印を描き、右端で同じ結果に合流するようにする。
- スタイル指定:
  - 日本語技術書向け
  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物
  - 複雑すぎる背景
  - 小さすぎる文字
  - 原典PDFの図に似せすぎること

## fig-ch32-partial-interface.png

- 対応箇所: 第32章「部分的なインターフェースでは足りない」
- 目的: IDだけを見る部分的なインターフェースでは、異なる内部値を区別できないことを示す。
- 画像生成プロンプト:
  左側に2つのカードを上下に置く。上のカードは「User A」「id=1」「name=Alice」、下のカードは「User B」「id=1」「name=Alicia」とする。中央に「公開インターフェース: getId」の細い窓を置き、両方のカードから窓へ矢印を引く。右側にはどちらも「1」と表示された同じ観測結果を置く。下部に薄く「name も公開すれば区別できる」と補足する余白を用意する。
