# 第23章 画像生成プロンプト

## fig-ch23-lossy-overview.png

- 対応箇所: 第23章 章冒頭
- 目的: lossy migration では、旧形式から新形式への変換で一部の情報が落ちるため、完全復元ではなく弱めた仕様を証明する、という章全体の構図を示す。
- 画像生成プロンプト:
  日本語の技術書に入れる白背景のフラットなベクター図。左に「旧形式 UserV1 / fullName」を置き、中央に「migrate」と書かれた右向き矢印、右に「新形式 UserV2 / firstName + lastName」を置く。旧形式の箱の中に「first」「middle」「last」の3つの小領域を描き、middle だけが薄い点線で下方向に落ちていく表現にする。下部に「完全復元ではなく、表示互換・条件付き復元・正規化後一致を証明する」という3つの小さなラベル領域を配置する。文字は後から差し替えやすい大きめのラベルボックスとして描く。
- スタイル指定:
  - 日本語技術書向け
  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物
  - 複雑すぎる背景
  - 小さすぎる文字
  - 原典PDFの図に似せすぎること

## fig-ch23-spec-weakening.png

- 対応箇所: 第23章「仕様弱化：証明可能な性質に落とす」
- 目的: 強すぎる仕様から証明可能な弱い仕様へ落とす階段状の関係を示す。
- 画像生成プロンプト:
  日本語の技術書向けの白背景ベクター図。左上に「強い仕様：完全 round-trip」と書ける大きな箱を置き、そこから下方向に階段状に「条件付き round-trip」「正規化後一致」「表示互換」と弱くなる仕様の箱を並べる。それぞれの箱の右側に、証明できる範囲を示す小さなメモ領域を付ける。強い仕様の箱には小さな警告アイコンを添え、下の仕様ほど安定して証明できる印象にする。色は淡い青・緑・グレー程度に抑える。
- スタイル指定:

  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物

## fig-ch23-normalization-quotient.png

- 対応箇所: 第23章「正規化として見る」
- 目的: 複数の旧データが、middle 情報を失って同じ新データへ写る様子を示し、lossy migration を商や抽象化の直感として理解させる。
- 画像生成プロンプト:
  日本語技術書用の白背景フラットベクター図。左側に3つの旧データの箱を縦に並べる。それぞれ「Ada Byron Lovelace」「Ada Augusta Lovelace」「Ada Lovelace」のように、middle だけが違う、または存在しないことを示すラベル領域を持つ。右側に1つの新データの箱「Ada Lovelace」を置く。左の3つの箱から右の1つの箱へ、合流する矢印を描く。矢印の上に「migrate / normalize」と書けるラベル領域を置く。下に「新形式では middle の差を観測しない」という注釈領域を置く。
- スタイル指定:
  - 日本語技術書向け
  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物
  - 複雑すぎる背景
  - 小さすぎる文字
  - 原典PDFの図に似せすぎること
