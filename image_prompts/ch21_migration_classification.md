- 目的: マイグレーションを一つの曖昧な「互換性」ではなく、完全同型、片方向互換、情報損失、制約付き同型、仕様弱化へ分類する視点を示す。

## fig-ch21-roundtrip-laws.png

- 対応箇所: 第21章「完全同型：情報を失わない変更」
- 目的: 完全同型では、旧形式から新形式へ移して戻す経路と、新形式から旧形式へ戻して移す経路の両方が恒等変換になることを視覚化する。
- 画像生成プロンプト:
  日本語の技術書に入れる、白背景のフラットなベクター図。左に「Old」、右に「New」と書ける二つの大きな箱を置く。Old から New へ「migrate」の矢印、New から Old へ「rollback」の矢印を描く。上側に Old -> New -> Old の戻り経路を描き、Old に戻ることを示すチェックマークを置く。下側に New -> Old -> New の戻り経路を描き、New に戻ることを示すチェックマークを置く。数式は最小限にし、ラベル領域を大きく取る。
- スタイル指定:

  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
