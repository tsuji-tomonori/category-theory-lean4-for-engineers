# 第3章 画像生成プロンプト

## fig-ch03-overview.png

- 対応箇所: 第3章「この章を読む理由」
- 目的: 関数の実行結果と、それを「等式（命題）」として読むことの関係を示す。
- 画像生成プロンプト:
  日本語の技術書に入れる白背景のフラットなベクター図。左に「関数 hasDiscount」の角丸ノード、中央に「実行結果 true / false」の小さなカード、右に「等式 hasDiscount true = true（主張）」のパネルを置き、左から右へ矢印でつなぐ。右下に「= は代入ではなく主張」という注記ラベルの配置スペースを作る。数式は最小限にし、余白を広く取る。
- スタイル指定:
  - 日本語技術書向け
  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物
  - 複雑すぎる背景
  - 小さすぎる文字
  - 原典PDFの図に似せすぎること
