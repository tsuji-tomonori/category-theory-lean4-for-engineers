# 第3章 画像生成プロンプト

## fig-ch03-overview.png

- 対応箇所: 第3章「この章を読む理由」
- 目的: 関数の実行結果と、それを「等式（命題）」として読むことの関係を示す。
- 画像生成プロンプト:
  日本語の技術書に入れる16:9横長、白背景のフラットなベクター図。左から「関数 hasDiscount」「評価 hasDiscount true」「実行結果 true」「等式 hasDiscount true = true（主張）」の順に角丸ノードを矢印でつなぐ。計算して値を得る前半と、得た値について命題を立てる後半を淡い背景色で分ける。下部に「= は代入ではなく主張」という注記を置く。文字列は指定どおり正確に一度だけ描き、章番号、図番号、図タイトルは画像内に入れない。数式は最小限にし、余白を広く取る。
- スタイル指定:
  - 日本語技術書向け
  - フラットなベクター図
  - 白背景
  - 余白多め
  - 色数は少なめ
  - 数式は最小限
  - 文字は後から差し替えやすい配置
- 避けること:
  - 写実的な人物
  - 複雑すぎる背景
  - 小さすぎる文字
  - 原典PDFの図に似せすぎること
